# Moments

A fully browser-based photo & video slideshow editor. Import media, arrange clips on a timeline, apply transitions and text overlays, add background music, and export a finished MP4 — no server required. All video encoding runs inside the browser via FFmpeg compiled to WebAssembly.

## Stack

| Layer | Tool |
|---|---|
| Framework | React 19 + Vite 8 |
| Video processing | @ffmpeg/ffmpeg 0.12 (WebAssembly / FFmpeg 5.1) |
| Icons | lucide-react |
| Styling | CSS Modules |
| State | Custom hook (`useMediaStore`) |

## Getting Started

```bash
npm install
npm run dev
```

Open http://localhost:5173

> The dev server sets `Cross-Origin-Opener-Policy` and `Cross-Origin-Embedder-Policy` headers required for `SharedArrayBuffer` (FFmpeg.wasm). This is already configured in `vite.config.js`.

## Features

### Media Panel (left)
- Drag & drop photos and videos, or click to browse
- Supports JPG, PNG, WEBP, GIF, HEIC, AVIF and MP4, MOV, WEBM, AVI, MKV, M4V
- Drag in an audio file (MP3, WAV, AAC, etc.) as background music with trim and volume control
- Video preview player with seekable progress bar

### Preview (center top)
- Live preview with all adjustments applied in real time
- Play/pause auto-advances through all clips with transitions
- Drag text overlays to reposition them

### Timeline (center bottom)
- Drag to reorder clips
- Transition pills between clips — click to change style or set to Cut (no transition)
- Global transition cycle button includes Cut for hard-cut-only exports
- Active clip highlighted with amber bottom bar
- Export duration chip shows trim-accurate encoded length

### Inspector (right)
- Clip duration, speed (video), brightness/contrast/saturation with live preview
- Image effects: Ken Burns, Pan & Zoom, Parallax, Fade In
- Blur background fill for letterboxed images
- Pan & zoom saved view per clip
- Text overlays: font, size, colour, animation, position (preset or drag)
- Custom font upload (.ttf, .otf, .woff)
- Transition type per clip (including Cut) and global default transition with duration
- Music volume and trim
- End fade (fade to black / fade out volume)

### About
- Click the **moments** logo in the top-left corner to open the About modal
- Displays app name, description, version, author, release date and repository link
- All fields are configurable via `public/app-info.json` — no rebuild required

### Export
- Quality: 480p (quick preview), 720p (default), 1080p
- Aspect ratio: 16:9 or 9:16
- Transitions: Crossfade, Slide Left, Slide Up, Zoom In, Dip to Black, or Cut (hard cut, zero re-encode)
- Text overlays burned in via FFmpeg drawtext
- Background music mixed at set volume with optional trim
- Optional end fade to black / audio fade out
- Downloads a single MP4

### Workflow
- Save your timeline, clips, text overlays, transitions, music settings, quality and volume as a JSON file
- Load it back in any session — media files are re-matched by filename
- Custom fonts are embedded in the workflow JSON (Base64) and restored automatically

### ⚠️ Required HTTP Headers

The following headers **must** be set on every response from your production server. Without them `SharedArrayBuffer` is blocked and FFmpeg.wasm will not load:

```
Cross-Origin-Opener-Policy: same-origin
Cross-Origin-Embedder-Policy: require-corp
```

**Nginx**
```nginx
add_header Cross-Origin-Opener-Policy "same-origin";
add_header Cross-Origin-Embedder-Policy "require-corp";
```

**Netlify** (`netlify.toml`)
```toml
[[headers]]
  for = "/*"
  [headers.values]
    Cross-Origin-Opener-Policy = "same-origin"
    Cross-Origin-Embedder-Policy = "require-corp"
```

**Vercel** (`vercel.json`)
```json
{
  "headers": [
    {
      "source": "/(.*)",
      "headers": [
        { "key": "Cross-Origin-Opener-Policy", "value": "same-origin" },
        { "key": "Cross-Origin-Embedder-Policy", "value": "require-corp" }
      ]
    }
  ]
}
```

### Notes

- The FFmpeg WASM files in `public/ffmpeg/` are ~32 MB. Ensure your host serves `.wasm` files and does not block them.
- Configure long-lived cache headers on `public/ffmpeg/` — these files rarely change and are expensive to re-download.
- The WASM binaries have been patched to pre-allocate 128 MB of initial heap (up from the default 32 MB). This is required for stable text overlay exports at 720p and 1080p. See `CLAUDE.md` for patch details.

## Export Pipeline Overview

The export runs in four stages inside a Web Worker via FFmpeg.wasm:

1. **Stage 1** — Each clip is encoded into a normalised `seg_N.mp4` (libx264, yuv420p, 30fps CFR, 1/90000 timebase, AAC 128k stereo).
   - **Images**: a single FFmpeg invocation handles blur background (via `split` filter), placement, optional effect/EQ, silence audio, and normalisation — all in one pass with no intermediate files.
   - **Videos**: placement + optional blur (via `split` filter) in one pass; audio extracted separately then muxed in the same terminal command; normalisation flags folded in.
   - Source files are written to WASM FS just-in-time (immediately before each clip's encode) and deleted immediately after, keeping peak heap usage low.

2. **Stage 2** — Segments are joined and music is mixed.
   - **All-cut exports** (all transitions = Cut, no music): concat demuxer stream-copy — zero re-encode.
   - **Single clip**: stream-copy to prefinal, optionally mix music.
   - **Mixed/animated transitions**: xfade + acrossfade filter chain. Hard cuts within a mixed graph use `concat` filter with immediate timebase normalisation (`settb=1/90000`) to prevent xfade crashes.

3. **Stage 3** — Text overlays burned in via FFmpeg `drawtext` in a single pass. Audio stream-copied.

4. **Stage 4** — Optional end fade to black / audio fade out. Output renamed via `ff.rename()` (zero-copy).

All FFmpeg commands run single-threaded (`-threads 1`). The WASM instance is terminated and recreated after every export. See `CLAUDE.md` for full pipeline details and critical constraints.
